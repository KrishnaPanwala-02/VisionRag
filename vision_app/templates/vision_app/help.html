🚀 Quick Start — 3 Steps

1

Upload a hardware image

Click the image button in the chat bar and pick a photo of your component — GPU, CPU, RAM, motherboard, monitor, etc. VisionRAG will identify it automatically using Gemini Vision.

2

Upload a PDF datasheet (optional but recommended)

Drop in the component's technical specification or manual PDF. VisionRAG will extract, chunk, and index it for retrieval. You can start chatting immediately — indexing continues in the background.

3

Ask anything

Type your question naturally. Ask follow-ups using "it", "that", or "the first one" — VisionRAG resolves references from context automatically.

📷 Uploading Images

✓

Supported formats

JPG, PNG, WebP, GIF, BMP, TIFF. Most phone and camera photos work out of the box.

✓

Blurry images are auto-sharpened

VisionRAG detects blur using Laplacian variance and applies an unsharp-mask sharpening pass before analysis. You'll see was_sharpened: true in the response when this happens.

✓

You can upload multiple images per session

Each image is identified and added as image context. All identified components are available when answering questions about specs.

📄 Uploading PDFs

✓

What kinds of PDFs work best?

Technical datasheets, user manuals, specification sheets, and white papers. Text-based PDFs work best. Scanned image-only PDFs will have limited text extraction.

✓

Multiple PDFs per session

Upload as many PDFs as you need. Retrieval runs across all of them simultaneously, with each chunk tagged by its source document.

✓

File size limit

Maximum upload size is 50 MB per file. For very large PDFs, consider splitting them into chapters for faster indexing.

💬 Chatting Effectively

→

Ask specific questions

Instead of "tell me about the GPU", try "what is the TDP of the GPU in the image?" Specific questions get more accurate retrievals.

→

Follow-ups work naturally

After asking "what are the power requirements?", you can say "and what about the cooling specifications?" — VisionRAG tracks the topic automatically.

→

Sessions are saved automatically

Every conversation is saved to your History. You can resume any session from the Dashboard or History page.

⚡ Pro Tips

🖼

Good lighting = better identification

Well-lit, in-focus images with the component label visible help Gemini Vision identify the exact component type and form factor.

📚

Upload the full datasheet, not just highlights

The full PDF gives the retrieval system more context to find the exact section relevant to your question.

🔄

Start a new session per component

Mixing components from different sessions can confuse retrieval. One session per hardware component keeps context clean.

🚀

Check provider status if responses are slow

Visit the Provider Status page to see which LLM is active and if any are rate-limited.

🔌 LLM Providers

VisionRAG automatically balances across multiple free-tier providers. If one hits its rate limit, the next takes over instantly.

G

Groq — Fastest (≈500 tok/s)

6,000 requests/day per API key. Get a key free at console.groq.com and add it to settings.py under GROQ_API_KEYS.

T

Together AI — $1 free credit

Roughly 1,000 requests. Add keys to TOGETHER_API_KEYS.

O

OpenRouter — 200 req/day (free models)

Add keys to OPENROUTER_API_KEYS. Slower but always available as a backup.

🦙

Ollama — Unlimited local fallback

Runs locally on your machine. Start with ollama serve and pull the model with ollama pull llama3.2. Used automatically when all cloud providers fail.
