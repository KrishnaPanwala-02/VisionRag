What is VisionRAG?

An AI-powered assistant that understands your hardware components from images and answers questions from your own documentation.

3 Advanced RAG Techniques Under the Hood
01

History-Aware Query Rewriting

Before searching, VisionRAG rewrites your question using the conversation history to resolve pronouns like "it" and "that" — so follow-up questions just work.

02

Reciprocal Rank Fusion

Two independent searches run simultaneously — keyword (BM25-style) and semantic vector — then fused with RRF. Neither alone is as good as both together.

03

Cross-Encoder Reranking

The top-15 candidates from RRF are reranked by a cross-encoder model that reads each (query, chunk) pair together — far more accurate than separate encodings.

How a Query Flows Through VisionRAG

Every question you ask goes through this pipeline automatically:

📷

1 · Upload a hardware image

Gemini Vision identifies the component — GPU, CPU, RAM, motherboard, etc. — and produces a structured label and description used as image context throughout the chat.

Gemini 2.0 Flash
📄

2 · Upload a PDF datasheet (optional)

Text is extracted and pre-chunked into overlapping 300-word windows. In the background, a ChromaDB vector store is built for semantic search. Chat works even before this completes.

PyPDF2 · ChromaDB · HuggingFace Embeddings
🔄

3 · Query rewriting

Your question is analysed for pronouns and short phrasing. If ambiguous, the last topic from conversation history is appended so retrieval gets the right context.

Zero-latency Python rewriter
🔍

4 · RRF retrieval

Keyword and semantic searches each produce a ranked list. RRF merges them using the formula score = Σ 1/(60 + rank) — chunks appearing high in both lists win.

BM25-style · Cosine similarity · RRF k=60
🎯

5 · Cross-encoder reranking

Top-12 candidates are re-scored by a cross-encoder that reads the full (question, chunk) pair together. The best 4 chunks go to the prompt.

ms-marco-MiniLM-L-6-v2
✨

6 · Answer generation

The final prompt combines image context, retrieved chunks, and recent chat history. The LLM answers from this grounded context only — no hallucinated training knowledge.

Groq · Together AI · OpenRouter · Ollama fallback
